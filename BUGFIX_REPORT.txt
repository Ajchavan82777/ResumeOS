A. BUGS FIXED (12 issues)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

BUG-01: Dashboard "Create Resume" silently failed
  Root Cause: Wrong response path. Code used `data.data.resume.id`
              but API returns `data.resume.id` (one level too deep).
  Fix:        dashboard/page.tsx — handle both response shapes safely.

BUG-02: Auto-save didn't persist section entries
  Root Cause: useAutoSave() only saved section metadata (title,
              visibility, sort_order) but never saved section_entries.
              All bullet points, job descriptions, and skill tags
              were lost on page reload.
  Fix:        useResume.ts — auto-save now iterates section_entries
              and calls sectionApi.updateEntry() for each one.

BUG-03: Drag-and-drop order not persisted
  Root Cause: reorderSections() updated Zustand state only.
              No API call was made to persist the new sort_order.
  Fix:        EditorPanel.tsx — onDrop() now immediately calls
              sectionApi.reorder() with the new order array.
              api.ts — added the missing `sectionApi.reorder()` method.

BUG-04: PDF export always fell back to window.print()
  Root Cause: exportApi.pdf() call in api.ts was missing
              `{ responseType: "blob" }`. Axios returned the response
              as JSON/text, the `instanceof Blob` check always failed,
              so it always called window.print() instead.
  Fix:        api.ts — added `responseType: "blob"` and `timeout: 60000`
              to exportApi.pdf(). BuilderShell.tsx — now opens a styled
              print-ready HTML window with instructions as reliable fallback.

BUG-05: DOCX export completely fake — no file downloaded
  Root Cause: Backend export.js returned `res.json({ resume, message })`
              — just JSON data, no file. Frontend BuilderShell.tsx called
              the endpoint and showed "DOCX ready!" toast with no download.
  Fix:        Created src/lib/exportDocx.ts — real client-side DOCX
              generation using the `docx` npm library (Document, Packer,
              Paragraph, TextRun, bullet points, tabs, page margins).
              Backend export.js — also generates server-side DOCX using
              docx package as primary method, sends binary buffer.
              BuilderShell.tsx — calls buildDocx() and uses file-saver
              saveAs() to trigger browser download.

BUG-06: Template switching didn't persist
  Root Cause: setTheme() only updated resume_customizations (accent_color,
              font etc.) but never saved template_slug to the resume record.
              After reload, the resume always reverted to its original template.
  Fix:        EditorPanel.tsx — handleTemplateChange() calls
              resumeApi.update(resumeId, { template_slug: slug }) immediately.
              useAutoSave also saves template_slug on each auto-save.

BUG-07: Manual "Save" button only saved customization
  Root Cause: handleSave() in BuilderShell.tsx only called
              resumeApi.updateCustomization() — sections and entries
              were not saved by the manual save button.
  Fix:        BuilderShell.tsx — handleSave() now saves customization,
              template_slug, all sections, and all section entries.

BUG-08: AI "Apply" button did nothing
  Root Cause: The "Apply to Resume" button in EditorPanel showed
              `toast.success("Copy the text above to apply it")` —
              this was a placeholder that never wrote anything to any field.
  Fix:        EditorPanel.tsx — applyAIResult() now:
              • summary → writes to summary section data.text
              • bullets → overwrites first experience entry bullets array
              • skills  → navigates to skills section with toast guide
              All changes trigger isDirty=true so auto-save picks them up.

BUG-09: No route guards — protected pages accessible without auth
  Root Cause: No middleware existed. Any URL was accessible without login.
  Fix:        src/middleware.ts — Next.js middleware checks for
              access_token cookie on all /dashboard, /builder, /settings,
              /admin routes. Redirects to /auth/login?redirect=<path>.
              useAuth.tsx — sets access_token cookie via document.cookie
              so middleware can read it.

BUG-10: No resume rename functionality
  Root Cause: No rename UI or API call existed anywhere.
  Fix:        dashboard/page.tsx — inline rename with pencil icon on hover.
              Click pencil → input appears inline → Enter to save →
              calls resumeApi.update(id, { title }) → refetches list.

BUG-11: No search or sort on dashboard
  Root Cause: Dashboard showed all resumes in insertion order only.
  Fix:        dashboard/page.tsx — search input filters by title/template.
              Sort dropdown: Last Modified / Date Created / Title (A-Z).
              Empty state when search returns no results.
              First-time empty state with CTA.

BUG-12: ATS score logic unreliable
  Root Cause: calcAtsScore() was checking section_entries.length > 0 but
              entries could be undefined when sections first load. Also
              lacked granular scoring for email vs phone separately, weak
              verb detection, and proper null guards.
  Fix:        atsScore.ts — complete rewrite with:
              • getEntries() helper with null guard
              • Granular contact scoring (name/email/phone separate)
              • Bullet count check (hasBullets)
              • Weak verb detection ("responsible for", "helped with")
              • More accurate breakdown percentages
